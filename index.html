Problem Description

Goals and Deliverables

Ideally, we plan to deliver an interactive experience that allows the user to manipulate a particle-based fluid within a bounding box and watch as it responds realistically to outside stimuli. The user will be able to click on the surface of the fluid to create ripples and drag their mouse to create waves. It is also a goal of ours to allow the user to rotate and change the shape/size of the box. We would like to implement relaistic shading via ray marching. In order to do so, we will need to familiarize ourselves with the platform we choose to build on as well as HLSL/OpenGL, which we will rely on for shaders.

We hope to deliver an application that runs at an acceptable framerate. If all goes well, we could extend the capabilities of our simulation to include things like foam, spray, and realistic refelction/refraction. When it comes to quantifying the quility/performance of our system, the framerate and number of particles being simulated will be key factor. Depending on the lighting system we implement, I imagine the number of rays used and the number of bounces calculated for each ray will be determining factors for performance.
